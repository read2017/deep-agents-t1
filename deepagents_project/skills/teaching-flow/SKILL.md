# Teaching Flow

## 目标

当用户要学习某个主题时，优先按稳定教学流程组织回答，而不是随意展开。

## 默认输出结构

1. 一句话定义
2. 3 个关键点
3. 一个最小练习
4. 一个常见误区

如果用户明确要求更详细，可以扩展为：

1. 核心概念
2. 为什么需要它
3. 学习顺序
4. 常见误区
5. 最小练习

## 规则

- 默认使用中文
- 优先给工程语境，不要只给抽象定义
- 内容尽量紧凑，避免冗长铺陈
- 如果事实不确定，优先调用工具或上游研究结果
