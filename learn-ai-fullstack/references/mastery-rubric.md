# 掌握度评估标准

## Level 1：知道

表现：

- 能复述概念定义
- 知道它大概做什么
- 能识别相关术语

仍然不算掌握的信号：

- 一离开定义就不会解释
- 只能背概念，无法举例

## Level 2：会用

表现：

- 能跟着示例完成基本实现
- 能在熟悉场景里使用该能力
- 能调通简单 demo

仍然不算掌握的信号：

- 稍微变个需求就不会
- 只能复制模板，不能解释原因

## Level 3：会改

表现：

- 能在已有项目里修改、扩展、修 bug
- 能解释当前实现为什么这样写
- 能处理常见边界情况

这是从“学生式理解”进入“工程式理解”的关键分水岭。

## Level 4：会设计

表现：

- 能从需求出发设计方案
- 能做基本技术选型与权衡
- 能划分模块、数据流和职责

判断问题：

- 为什么选这个方案而不是另一个
- 哪些地方以后会成为维护成本

## Level 5：会迁移

表现：

- 能把同一能力迁移到不同项目或业务场景
- 能识别背后通用模式
- 能指导别人或给出 review 意见

这是“真正掌握”的标志。

## 使用方法

评估时不要只用一个问题下结论。至少组合两种证据：

- 口头解释或文字复述
- 小实现
- 改错题
- 变式题
- 应用于用户自己的项目

如果用户只达到 `知道` 或 `会用`，不要默认已经掌握。  
如果用户能独立修改、设计并迁移，才算进入稳定掌握。
