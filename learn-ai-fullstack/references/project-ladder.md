# 项目阶梯

## 设计原则

- 每个项目只锻炼少数关键能力
- 优先贴近 AI 教育产品场景
- 从“功能能跑”逐步升级到“结构清晰、能上线、可维护”

## Level 1：微项目

适合单个知识点刚学完后的巩固。

### 1. 学习计划生成器

能力点：

- 前端表单
- API 调用
- FastAPI 基础接口

交付物：

- 输入学习目标、当前水平、可投入时间
- 输出结构化学习计划

### 2. 概念问答小助手

能力点：

- Prompt 基础
- 前后端联调
- 错误处理和加载状态

交付物：

- 一个可以就单个知识点进行问答的简单页面

### 3. 学习任务看板

能力点：

- Vue 3 状态管理
- CRUD
- 列表与筛选

交付物：

- 可以新增、编辑、标记完成学习任务的页面

## Level 2：中型项目

适合一个阶段结束后的整合训练。

### 1. AI 学习陪练后台

能力点：

- Vue 3 页面组织
- FastAPI 服务分层
- 数据库存储
- 用户输入与反馈链路

交付物：

- 用户可以提交学习问题
- 系统给出解释、练习、下一步建议
- 保存学习记录

### 2. 教育知识库 RAG 应用

能力点：

- 文档切分
- 向量检索
- 检索增强生成
- 评测基础

交付物：

- 上传教育领域知识材料
- 支持带来源回答
- 能调试召回质量

### 3. 顾问工作流助手

能力点：

- 多角色数据建模
- 任务流转
- LLM 辅助生成

交付物：

- 支持顾问管理学生、任务、沟通记录和建议草稿

## Level 3：综合项目

适合冲刺“AI 全栈专家”方向的阶段作品。

### AI 教育 Copilot

建议模块：

- 学习画像与目标录入
- 学习路径规划
- 知识问答
- 任务拆解与跟踪
- 顾问后台
- 基础评测与日志

能力点：

- 全栈联调
- LLM 工作流编排
- 数据建模
- 结构化输出
- 部署运维

验收标准：

- 用户能完成完整学习闭环
- 前后端边界清晰
- 有基础错误处理、日志和部署方案
- 至少有一个模块能体现 AI 相比普通 CRUD 的真实价值

## 升级方式

当用户完成当前项目后，可以沿这些方向升级：

- 增加用户体系和权限
- 增加历史记录与可观测性
- 加入评测与反馈闭环
- 引入异步任务和队列
- 做部署与线上可用版本
