# AI 全栈学习路线

## 路线原则

- 先补齐 Web 与工程基础，再放大到 LLM 应用
- 每一阶段都要有可以交付的项目
- 每一阶段的终点不是“看懂”，而是“能独立做出来”

## 阶段 1：JavaScript 与现代前端基础

目标：能看懂并修改现代前端项目，理解组件、状态、异步、模块化。

重点：

- JavaScript 语法与执行模型
- ES modules
- Promise、async/await
- DOM 与浏览器基本机制
- npm、包管理、脚本、环境变量

最低项目：

- 做一个原生 JS 小工具页
- 改造一个已有前端 demo

进入下一阶段标准：

- 能解释闭包、事件循环、模块导入导出
- 能读懂常见异步代码
- 能独立调试简单前端报错

## 阶段 2：Vue 3 应用开发

目标：能独立写出结构清晰的 Vue 3 页面和基础业务流。

重点：

- Composition API
- `ref`、`reactive`、`computed`、`watch`
- 组件通信
- 表单与状态管理
- 路由、页面组织
- 与后端 API 交互

最低项目：

- 学习任务管理页面
- AI 教育咨询后台的单页模块

进入下一阶段标准：

- 能独立拆组件
- 能处理表单、列表、详情、加载状态、错误状态
- 能解释响应式和组件生命周期

## 阶段 3：FastAPI 与 Python Web 后端

目标：能设计和实现后端 API，支撑前端真实业务。

重点：

- 路由、请求/响应模型
- Pydantic 数据校验
- 依赖注入
- 异步接口
- 服务分层
- 配置管理、日志、错误处理

最低项目：

- 学习计划生成 API
- 用户问答记录 API

进入下一阶段标准：

- 能解释依赖注入和 schema 的作用
- 能独立写 CRUD 和业务接口
- 能定位常见 422、导入、协程相关报错

## 阶段 4：数据库、鉴权与 API 工程化

目标：能把前后端拼成完整业务系统，而不是孤立 demo。

重点：

- SQL 基础与 ORM
- 数据建模
- 用户体系与权限
- API 约定
- 文件上传
- 缓存、任务队列、基础测试

最低项目：

- 学生/顾问/任务三类实体的教育工作流系统

进入下一阶段标准：

- 能从业务需求反推数据模型
- 能解释接口边界、错误码与权限控制
- 能让前后端完整联调

## 阶段 5：LLM 应用开发

目标：能把模型能力接入业务，而不是只会调用一次 API。

重点：

- Prompt 设计
- 结构化输出
- 工具调用
- RAG
- 工作流编排
- 基础评测与可观测性

最低项目：

- AI 学习问答助手
- 面向教育场景的知识库检索应用
- 学习规划 Agent 或工作流

进入下一阶段标准：

- 能解释为什么需要 RAG / workflow / eval
- 能设计简单的知识库问答链路
- 能区分原型可用和生产可用

## 阶段 6：部署、运维与产品化

目标：能把系统跑起来、上线、维护、迭代。

重点：

- Docker
- 环境配置
- 部署平台
- 基础监控与日志
- 成本意识
- 故障排查

最低项目：

- 将完整 AI 教育产品 demo 部署到可访问环境

进入下一阶段标准：

- 能独立部署前后端与模型服务
- 能处理基础线上问题
- 能说清楚成本、性能、稳定性的取舍

## 推荐节奏

- 每周只攻一个核心主题
- 每两周至少交付一个小项目
- 每四到六周完成一个阶段性整合项目
